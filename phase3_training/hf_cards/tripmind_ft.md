# pivotai-ft

Fine-tuned Llama 3.1 8B for Indian domestic travel optimization. Given a traveler persona, generates an optimized day-by-day itinerary identifying **Price-Pivot Points** — transit, accommodation, or activity substitutions that save ≥5% without degrading trip quality.

Part of the [pivotai](https://github.com/ishreyadev/pivotai) project: a multi-agent AI travel optimizer trained via three distinct approaches (SFT, distillation, curriculum). **pivotai-ft** is the best-performing variant, trained via standard supervised fine-tuning on 5,000 synthetic pairs generated by GPT-4o-mini.

## Model Details

| Property | Value |

| Training method | QLoRA r=8, α=16, dropout=0.05 |

| Epochs | 3 |

## Usage with Ollama

```bash
# Download GGUF from this repo
ollama create pivotai-ft -f Modelfile.ft

# Run
ollama run pivotai-ft
```

## Limitations

- Trained on Indian domestic travel only (20 cities). Not designed for international travel.
- Red-team robustness is below target (53.3% vs 80% goal) — the model can be prompted to bypass budget constraints.
- Intent alignment is below target (32.2% vs 55%) — cost optimization is prioritized over activity personalization.
- Inference on CPU takes 30–120 seconds per query (use GPU for production).
